小组分工

鉴于大家基础不是很好，所以我们首先先试行这样的分工（麻雀虽小 五脏俱全）。

  1. 为了提高大家综合能力，咱们每个人都属于每个小组
  2. 咱们一共5个人，所以有下面这5个分工
    1. 产品部
    2. 后端部
    3. 前端部
    4. UI部
    5. 统筹部
  3. 每个部的部长专门负责这他部门一块。每个部门都很重要。

下面对这几个部门，做单独的分工说明：

产品部

  1. 前期素材收集，这要求，平时要多多留意漂亮的图片，还要求有一定的ps能力（去除水印呀，什么的）收集素材记得要保留来源，比如 网址啦，或者其他（版权）。
  2. 市场调研，市场预测。
  3. 策划。
  4. 项目进度监督。
  5. 后期消费群体推广。
  6. 公司公共关系。（提升公司形象）

  7. 新晋人员培训、考核。

    技能需求：

    1. ps
    2. 有一定的逻辑思维能力，能做项目进度报告
    3. 眼界开放，能长远的思考一些问题。
    4. 需要点公关知识

UI部

  1. 项目人机交互
  2. 项目操作逻辑
  3. 项目界面美观
  4. 项目整体设计

  5. 人性化思想（最终为软件用户方便使用软件着想）。

    技能需求：

    1. 熟悉使用 Axure RP 8
    2. 需要要有人性化服务的思想
    3. 平时需要多多观察优秀网站和app学习别人优秀的交互
    4. 要有发现美的眼光

后端部

  1. 后台搭建（MVC框架）（暂时不考虑使用流行的框架，就自己搭建了一个简单的框架，这样所有的操作，都由我们具体实现，对我们理解MVC框架思想有很大的帮助）。
  2. 数据库搭建与维护。
  3. 根据项目需求提供给前端对应端口。

  4. 服务器的搭建与维护

    技能需求：

    1. 逻辑性强
    2. 熟悉mySQL数据库表的建设（事务，依赖关系，存储过程等等专业知识）
    3. 较强的编程能力
    4. 咱们项目首先推荐使用的是php作为后台
    5. 有点http的相关知识，清晰的明白一个请求的发出与结束的过程

前端部

  1. 项目人机交互实现
  2. 项目操作逻辑实现
  3. 项目界面美观实现

  4. 项目人性化数据展示的实现

    技能需求：

    1. 熟悉使用 webstorm。
    2. 熟悉html。
    3. 熟悉css。
    4. 熟悉js。
    5. 咱们是基于angular的ionic框架编写的，要求比较高了点（因为这里是最核心的，这里的好坏，是用户能直接感受到的）。
    6. 熟悉angular框架的使用
    7. 熟悉ionic框架的使用

统筹部

  1. 项目糅合
  2. 项目制作流程的记录
  3. 项目讨论记录

  4. 项目贡献记录

    技能需求：

    1. 熟悉web全栈开发。

基础技能要求

就是说要了解点

  1. WebStorm（从官网下正版，我有密钥）
  2. Emmet（WebStorm自带，这是一门快速编程的语言）
  3. Git(必须熟练，可以网上搜索下，我也会提供给大家点资料，版本控制器)
  4. markdown（超级方便的书写方法）
  5. html
  6. css
  7. js
  8. mySQl
  9. php
  10. Axure RP（UI设计软件）
  11. Json（前后端数据交互）
  12. AJAX
  13. jQuery（这些比较高深了，有兴趣可以看看）
  14. bootstrap
  15. angular
  16. Sass与Compass
  17. ionic
  18. 项目课余时间做，现在学习还是最重要的，不要本末倒置。
